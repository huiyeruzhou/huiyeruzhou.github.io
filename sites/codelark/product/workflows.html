核心工作流 ​

接管已有本地会话 ​

用户在 IM 里发送 /t，bridge 会列出本机可发现的 Codex / Claude Code 会话。用户再发送 /t 1 或点击卡片按钮，就可以把当前聊天绑定到对应本地会话。

用户价值：

  • 不需要复制 thread id。
  • 能从 IM 里看到本机最近会话。
  • 绑定后直接发消息即可继续同一条会话。

设计模块：

  • 本地 runtime 会话发现：src/operator-ui/application/session-source.ts
  • Codex session 索引：src/runtime/codex/session-index.ts
  • Claude Code JSONL 发现：src/runtime/claude/session-jsonl.ts
  • 线程命令：src/bridge/command/session-thread.ts

在 IM 中继续对话 ​

聊天绑定到 BridgeSession 后，普通文本会进入当前 runtime。Codex 支持 sdk、pty、tmux provider；Claude Code 支持 pty、sdk provider。

用户价值：

  • 同一个 IM 聊天可以从 Codex 切到 Claude Code。
  • 会话级模型、工作目录、权限模式、sandbox、network 和 reasoning 可以独立覆盖。
  • provider 底层差异被隐藏在统一命令和会话模型后面。

设计模块：

  • 普通消息进入 runtime：src/bridge/turn/interactive/runner.ts
  • runtime/provider 命令：src/bridge/command/runtime-settings.ts 和 src/bridge/command/provider-settings.ts
  • provider 路由：src/runtime/codex/routing-provider.ts

创建新的 IM 群聊会话 ​

用户发送 /new 或 /new <名称> <路径>，bridge 会基于当前会话目录创建新的 IM 群聊会话，并把新群绑定到新的 BridgeSession。平台支持时，群名会同步体现用户指定的名称。

设计模块：

  • /new 和 /clear：src/bridge/command/session-thread.ts
  • 表单卡片：src/bridge/command/presentation/index.ts
  • 群聊创建/改名能力：src/channels/contracts.ts
  • 飞书实现：src/channels/feishu/adapter.ts

观察本地会话输出和 mirror ​

当本机 Codex TUI、Codex Native 或 Claude Code 在同一条本地会话里继续输出时，mirror 机制会读取对应 JSONL，把新增 turn 同步到 IM。这样 IM 不只看到自己发起的 prompt，也能看到本地客户端继续产生的结果。

设计模块：

  • mirror 订阅：src/bridge/mirror/subscription-registry.ts
  • mirror runtime：src/bridge/mirror/runtime.ts
  • turn 合并：src/bridge/mirror/turns.ts
  • 卡片反馈：src/bridge/mirror/feedback-controller.ts

使用终端、shell 和文件能力 ​

Codex-to-IM 支持几类面向开发工作的远程工具：

  • /shell [秒数] <command> 在当前会话目录用 codex sandbox 流式执行命令。
  • /tmux-* 查看、绑定、新建 tmux session，并发送按键。
  • /pty-screen 查看当前 pty runtime 的屏幕输出。
  • /cat <path> 打印文件片段。
  • /file <path> 把本地文件发回 IM。

设计模块：

  • shell 命令：src/bridge/command/shell.ts
  • tmux 命令：src/bridge/command/tmux.ts
  • pty 屏幕：src/bridge/command/pty.ts
  • 文件和附件：src/channels/delivery/artifacts.ts
  • 输入安全校验：src/shared/security/validators.ts

自动任务 ​

用户可以通过 /auto 创建基于时间间隔的自动 prompt，也可以通过 /auto-script 创建脚本驱动的自动任务。脚本任务把 stdout 作为下一轮当前 session runtime prompt。

设计模块：

  • 自动任务模型：src/bridge/automation/auto-tasks.ts
  • 自动任务命令：src/bridge/command/auto.ts
  • 自动任务调度：src/bridge/host/manager.ts
  • 自动脚本 skill：skills/codex-to-im-auto/SKILL.md
