产品定位与边界 ​

一句话定位 ​

Codex-to-IM 是本机 bridge，不是云端托管服务。它把本机 Codex / Claude Code 会话接入飞书 IM 通道，让用户在聊天软件里继续操作本地 coding session。

目标用户 ​

  • 远程操作型用户：离开本机 IDE 后，仍希望在飞书里继续某个会话。
  • 团队协作型用户：希望把一个本地 coding session 接入群聊，团队成员能看到进度、检查结果、触发后续命令。
  • 本地自动化用户：希望通过 IM 触发诊断、历史查看、shell、定时 prompt、文件回传等工作流，同时保持配置和日志仍在本机。

Codex-to-IM 负责什么 ​

  • 接收 IM 消息，识别命令或普通 prompt。
  • 把聊天绑定到本地 BridgeSession。
  • 按会话配置选择 Codex 或 Claude Code 运行路径。
  • 把流式输出、工具调用、权限请求、历史、附件和诊断结果发回 IM。
  • 维护本地配置、通道实例、绑定、会话缓存和审计日志。

Codex-to-IM 不负责什么 ​

  • 不托管用户代码仓库。
  • 不代替 Codex / Claude Code 管理账号登录态。
  • 不改写 ~/.codex 或 Claude Code 自有会话格式。
  • 不把本地会话同步到远端数据库。

产品概念 ​

概念用户视角开发者视角
BridgeSession一条可持续的本地对话Codex-to-IM 自有 session 容器
ChannelChat某个飞书群聊或单聊当前接到哪条会话IM chat 到 BridgeSession.id 的绑定
IMChannel一个飞书机器人入口adapter 实例和平台配置
Runtime当前用 Codex 还是 Claude Codeprovider 路由的第一层
Provider用 SDK、pty 还是 tmux 运行provider 路由的第二层

详细业务模型见 当前架构。

设计模块 ​

  • 类型和通道地址：src/domain/index.ts
  • 通道抽象：src/channels/contracts.ts
  • 会话绑定：src/bridge/session/channel-router.ts
  • 会话注册表：src/bridge/session/registry.ts
  • 本地存储：src/storage/json-store.ts
