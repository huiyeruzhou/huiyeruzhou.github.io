Codex 工具调用展示设计 ​

目标 ​

Codex-to-IM 已经把 SDK 事件和本地 Codex session mirror 事件统一到内部 turn 流程。工具调用展示也必须遵守同一个原则：SDK 和 mirror 不能各走各的解析与渲染逻辑。

这次实现的目标是：

  • 在 turn 解析阶段保留工具调用的结构化语义，而不是把 input/output 过早压平成短字符串。
  • 在 reducer 阶段用同一套规则合并 SDK 和 mirror 工具状态。
  • 在推出到 IM、streaming card、final card、session history 时复用同一套工具渲染函数。
  • 对 Codex 常见工具做结构化展示，尤其是 exec_command、write_stdin 和 apply_patch。
  • 默认隐藏 call_id、tool_use_id、Chunk ID、Original token count 等内部关联字段。
  • 不在工具 detail 主路径截断 stdout/stderr/聚合输出；平台消息长度限制应由 delivery 层处理。

统一管线 ​

工具展示现在分成四层：

  1. source parser

    SDK SSE 路径和 mirror JSONL 路径分别从上游事件读取工具 input/output，但只负责抽取事实，不负责拼 UI 文案。

  2. tool detail parser

    src/shared/progress/tool-call-details.ts 把工具 input/output 解析为 CodexToolDetail。这是 SDK、mirror、history 共用的结构化语义层。

  3. turn reducer

    src/runtime/codex/turn-events.ts 把 CodexTurnEvent 合并到 ToolCallInfo。它不再只保存 input / output 字符串，还保存 detail。

  4. renderer

    src/shared/progress/tool-rendering.ts 负责把 ToolCallInfo 渲染成共享 Markdown / progress block。Feishu streaming/final card 和 SDK inline preview 都走这层；Feishu adapter 只负责把共享 block 包装成 CardKit collapsible_panel，并保留长 exec_command 输出的第二层折叠。

核心类型 ​

ToolCallInfo 仍然保留旧字段，保证现有 adapter 和 fallback 展示兼容：

ts
export interface ToolCallInfo {
  id: string;
  name: string;
  status: 'running' | 'complete' | 'error';
  input?: string | null;
  output?: string | null;
  detail?: CodexToolDetail | null;
}

detail 是新的事实来源。当前覆盖这些工具形态：

  • exec_command
  • terminal_stdin
  • patch_apply
  • tool_search
  • web_search
  • mcp
  • dynamic
  • generic

统一 reducer 始终保存 input、output 和 detail。历史上的 showToolCallDetails 开关已移除，不再作为底层工具事实保存或渲染的分支条件。

工具展示规则 ​

exec_command / Bash ​

输入解析：

  • 优先读取 cmd。
  • 兼容读取 command。
  • 兼容读取 args / argv 数组。
  • 如果命令是 /bin/bash -lc "<cmd>"，展示时剥掉 wrapper。
  • 展示 workdir、shell、tty 这类有助于理解执行环境的 metadata。

输出解析：

  • 从 Wall time: ... seconds 解析耗时。
  • 从 Process exited with code N 或 ExecToolCallOutput { exit_code: N } 解析退出码。
  • 从 Process running with session ID N 解析仍在运行的 PTY session。
  • 从 Output: 后提取真正聚合输出。
  • 隐藏 Chunk ID、Original token count 等内部元数据，只展示真正输出内容。
  • 从 sandbox/CreateProcess failure 文本里尽量保留可读错误输出。

展示格式：

markdown
workdir: `/sites/codelark/repo/path`

```bash
npm test
```

```text
tests passed
```

成功且退出码为 0 时，正文不再重复展示 Success with exit code 0；耗时展示在工具面板标题里，例如 ✅ `exec_command` · 完成 · 1.2s。失败或非 0 退出码仍在正文保留状态行。

失败时使用警示标记：

text
⚠️ Fail with exit code 191 in 320ms.

在 Feishu streaming/final card 中，exec_command 的短输出仍在工具详情内用 text fence 展示；过长输出会从主工具详情中移出，放进工具面板内的第二层折叠面板，避免一次工具调用把整张卡片撑得过长。

write_stdin ​

输入解析：

  • session_id 展示为 session 标签。
  • chars 非空时用多行代码块展示。
  • chars 为空时显示为读取终端输出，不渲染空代码块。

输出解析与 exec_command 使用同一套进程状态解析，但不把 write_stdin 强行关联回上一个 exec_command。它是独立的一次终端读写事件。

apply_patch ​

输入解析：

  • 从 patch header 解析文件操作摘要：add / update / delete / move。
  • 完整 patch 作为 diff code fence 展示。
  • 兼容裸 patch 字符串和结构化输入对象；当对象字段 patch / diff / input / cmd / command 内含 *** Begin Patch 时，优先抽取该字段作为 patch 文本，避免把整段对象渲染成 json fence。

输出解析：

  • mirror 的 patch_apply_end.changes 优先作为文件摘要。
  • 没有结构化 changes 时展示 stdout/stderr。
  • 输出完整保留。
  • 输出使用 diff fence，便于文件变更摘要和 patch 失败信息高亮。

飞书原生折叠面板用于流式卡片和最终卡片里的工具详情。面板必须使用 JSON 2.0 的 collapsible_panel，并遵守飞书 element_id 约束：同卡唯一、字母开头、只包含字母数字下划线，且不超过 20 字符。当前工具面板 ID 使用 stream_tool_N。

折叠面板标题支持 plain_text 和 markdown。工具名标题需要保留反引号代码样式时，必须显式启用 markdown 标题：

json
{
  "tag": "collapsible_panel",
  "element_id": "stream_tool_1",
  "expanded": false,
  "header": {
    "title": {
      "tag": "markdown",
      "content": "✅ `Bash` · 完成"
    },
    "template": "green"
  },
  "elements": [
    {
      "tag": "markdown",
      "content": "工具详情",
      "text_size": "notation"
    }
  ]
}

活跃目标 ​

mirror 里的 leading Goal Active 会先从正文拆出，再单独渲染。短 goal 使用 notation markdown 元素；超过阈值的长 goal 使用折叠的 collapsible_panel，标题保留压缩后的目标摘要，展开后展示完整 objective。该逻辑同时用于 streaming card 和 final card，避免目标内容在正文里重复出现。

不要把带 Markdown 标记的标题写成 plain_text，否则飞书会把反引号等标记作为普通字符显示。

update_plan ​

update_plan 不作为普通工具展示。它继续进入 task progress 区域。

目标工具 ​

get_goal / create_goal / update_goal 默认走 generic fallback。展示时不特殊展开 threadId、objective 等内部长上下文。

tool_search ​

输入展示 query。

输出展示：

  • 找到的工具数量。
  • namespace 列表。
  • 前几个工具名。

不展开完整工具 schema。

Web 搜索 ​

Web Search 默认不展开细节。上游通常只给 query 或完成信号，把它铺开会比实际信息更噪。

MCP ​

当前上游 MCP 信息不完整。实现策略是保留 namespace：mcp__<server>__<tool>，能解析 arguments/result/error 时展示；没有 result 时不制造假输出。

动态工具 ​

展示 tool 名、arguments、content_items 或 error。mirror 的 dynamic_tool_call_request/response 和 SDK 侧未来相同形态都应进入同一种 detail。

通用兜底 ​

未知工具使用 fallback：

  • input 用 JSON pretty print。
  • output 优先展示文本；无法识别时展示 JSON。
  • 默认不展示 call id。

接入点 ​

SDK 流式事件 ​

文件：

  • src/bridge/turn/interactive/sdk-conversation-engine.ts
  • src/bridge/turn/interactive/sdk-stream-events-controller.ts
  • src/runtime/codex/turn-events.ts

SDK 的 tool_use / tool_result 仍然进入 onToolEvent，但 detail 解析集中在 codex-turn-events -> tool-call-details。

Mirror JSONL ​

文件：

  • src/runtime/codex/session-index/event-mirror-parser.ts
  • src/bridge/mirror/turns.ts
  • src/runtime/codex/turn-events.ts

mirror parser 在生成 CodexMirrorRecord 时附带 toolDetail，mirror turn reducer 再走同一个 applyCodexTurnEventToTools。

event-mirror-parser.ts 不再分别手写 function/custom/tool_search/dynamic/exec/apply_patch 的 detail 拼装；这些 JSONL 工具事件先进入 src/runtime/codex/session-index/tool-call-events.ts 转成 CodexTurnEvent，再由同一个事件事实派生 mirror record。这样 SDK 流和 mirror 流的工具状态共享 CodexTurnEvent / CodexToolDetail 语义。

会话历史 ​

文件：

  • src/runtime/codex/session-index/history-parser.ts

history parser 维护一个轻量 call_id -> tool state map，用来把 function_call 和 function_call_output 合并成同一种 detail，然后复用 renderCodexToolDetailMarkdown。

IM 卡片渲染 ​

文件：

  • src/channels/feishu/markdown.ts
  • src/bridge/turn/interactive/sdk-stream-preview.ts
  • src/shared/progress/tool-rendering.ts

Feishu streaming/final card 和 SDK inline preview 都通过 buildToolProgressMarkdown / buildToolProgressBlocks 渲染 ToolCallInfo。renderCodexToolDetailMarkdown 仍负责单个 CodexToolDetail 的正文片段，但不会再由 SDK inline preview 自己判断 bash/json/diff fence。

验证 ​

覆盖范围：

  • exec_command 的 cmd / workdir 展示。
  • exec_command 文本输出只展示 Output: 后的聚合输出，不泄露 Chunk ID / Original token count 等元数据。
  • Success/Fail with exit code ... in ... 状态行。
  • 聚合输出展示。
  • apply_patch 裸 patch 和对象内 patch 都使用 diff fenced block。
  • apply_patch 输出使用 diff fenced block。
  • exec_command 非 0 退出在标题和正文显示警示标记。
  • exec_command 长输出在 Feishu 工具面板内使用第二层折叠面板。
  • 长 active goal 在 streaming/final card 中使用折叠面板展示，正文不重复 goal。
  • showToolCallDetails 用户可见开关已移除，统一 reducer 始终保留 input、output、detail。
  • SDK 和 mirror reducer 都生成同样的 ToolCallInfo.detail。
  • SDK inline preview 和 Feishu 工具卡片共享 ToolCallInfo -> Markdown/block 渲染入口。
  • Feishu tool markdown 使用结构化 detail，不再展示 ignored raw input/output。
  • Codex session history 与 mirror parser 保持主要工具事件可读。
